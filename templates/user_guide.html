User Guide for Microservices Application

A prototype API for a music microservice.

Through the beauty of Makefiles, we can initialize and spin up all four microservices with one command!

In the terminal, run: make init

This will run two commands for you. It runs

  1. flask init to initialize the database
  2. foreman start to spin up our four microservices.

Now that the database is set up and the four services are up and running, feel free to test the various routes. Or, see how to run the shell scripts.

How to use the shell script(s):
  • To perform a series of POST requests using a shell script to populate all of our API endpoints , run: make seedapi
Track Microservices:
  • Create a new track, send a POST request in JSON format to:
    http://127.0.0.1:5001/api/v1/resources/musicService/tracks

    • JSON must have these fields:
    • track_title
    • album_title
    • artist
    • length_seconds
    • url_media
    • url_art(Optional)

    Example of a POST curl command:

    curl -v -d '{"track_title" : "MYSONG", "album_title": "MYALBUM", "artist" : "BestArtist", "length_seconds" : "201", "url_media" : "wwww.soundcloud.com/thisSong", "url_art" : "wwww.flickr.com/thisImage"}' -H "Content-Type: application/json" -X POST http://127.0.0.1:5001/api/v1/resources/musicService/tracks


  • Retrieve a track, go to URL:
    http://127.0.0.1:5001/api/v1/resources/musicService/tracks?track_title=track_title

    Replace track_title at end of URL with title of track you'd like to retrieve.
    • If track_title is not known retrieve all tracks by going to URL:
      • http://127.0.0.1:5001/api/v1/resources/musicService/tracks/all


  • Edit a track, send a PUT request in JSON format to:
    http://127.0.0.1:5001/api/v1/resources/musicService/tracks/edit-track

    • JSON must have these fields:
    • track_title
    • artist
    • newTrackTitle
    • newAlbumTitle
    • newArtist
    • newLength
    • newUrlMedia
    • newUrlArt

    Example of a PUT curl command:

    curl -v -d '{"track_title" : "Stan", "artist": "Eminem" , "newTrackTitle" : "NEWSONG", "newAlbumTitle": "NEWESTALBUM", "newArtist" : "NEWBestArtist", "newLength" : "180", "newUrlMedia" : "wwww.soundcloud.com/thisSong", "newUrlArt" : "wwww.flickr.com/thisImage"}' -H "Content-Type: application/json" -X PUT http://127.0.0.1:5001/api/v1/resources/musicService/tracks/edit-track


  • Delete a track, send a DELETE request in JSON format to:
    http://127.0.0.1:5001/api/v1/resources/musicService/tracks

    • JSON must have these fields:
    • track_title
    • artist

    Example of a DELETE curl command:

    curl -v -d '{"track_title" : "Sunflower", "artist": "Post Malone"}' -H "Content-Type: application/json" -X DELETE http://127.0.0.1:5001/api/v1/resources/musicService/tracks


User Microservices:
  • To Create a User, send a POST request in JSON format to: http://127.0.0.1:5000/api/v1/resources/musicService/users
    Post must have these fields:
    • username
    • password
    • display_name
    • email
    • homepage_url (Optional)

  • To Retrieve a Profile, go to URL: http://127.0.0.1:5000/api/v1/resources/musicService/users?username=user
    Replace user at end of URL with username of user you'd like to retrieve.

  • To Delete a User, send a DELETE request in JSON format to: http://127.0.0.1:5000/api/v1/resources/musicService/users
    Post requires a username field.

  • To Change a User's Password, send a PUT request in JSON format to: http://127.0.0.1:5000/api/v1/resources/musicService/users/change-password
    Post must have these fields:
    • username
    • newpassword

  • To Authenticate a User, send a POST request in JSON format to: http://127.0.0.1:5000/api/v1/resources/musicService/users/authenticate-user
    Post must have these fields:
    • username
    • newpassword
Description Microservices:
  • To Set a User's Description of a Track, send a POST request in JSON format to: http://127.0.0.1:5002/api/v1/resources/musicService/descriptions
    Post must have these fields:
    • username
    • track_id
    • description_text

  • To Retrieve a User's Description of a Track, go to URL:
    http://127.0.0.1:5002/api/v1/resources/musicService/descriptions?username=username&track_id=track_id
    Replace username at end of URL with username of user's description you'd like to retrieve and replace track_id of track_id of the specific track.
Playlist Microservices:
  • Create a new playlist, send a POST request in JSON format to:
    http://127.0.0.1:5003/api/v1/resources/musicService/playlists

    • JSON must have these fields:
    • playlist_title
    • description
    • username_id

    Example of a POST curl command:

    curl -v -d '{"playlist_title" : "The Feels Train", "description" : "Curl up and cry.", "username_id" : "ian123"}' -H "Content-Type: application/json" -X POST http://127.0.0.1:5003/api/v1/resources/musicService/playlists


  • Retrieve a playlist, go to URL:
    http://127.0.0.1:5003/api/v1/resources/musicService/playlists?playlist_title=playlist_title

    Replace playlist_title at end of URL with title of playlist you'd like to retrieve.
    • If playlist_title is not known retrieve all playlists by going to URL:
      • http://127.0.0.1:5003/api/v1/resources/musicService/playlists/all


  • Delete a playlist, send a DELETE request in JSON format to:
    http://127.0.0.1:5003/api/v1/resources/musicService/playlists

    • JSON must have these fields:
    • playlist_title

    Example of a DELETE curl command:

    curl -v -d '{"playlist_title": "The Feels Train"}' -H "Content-Type: application/json" -X DELETE http://127.0.0.1:5003/api/v1/resources/musicService/playlists


  • List all playlists, go to URL:
    • http://127.0.0.1:5003/api/v1/resources/musicService/playlists/all

  • Lists playlists created by a particular user
    • http://127.0.0.1:5003/api/v1/resources/musicService/playlists/user?username_id=ian123
